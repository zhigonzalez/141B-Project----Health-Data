# Diabetes and Heart Disease Data Cleaning and Analysis Project

## Project Overview
This project analyzes data related to diabetes and heart disease, conducting data cleaning, basic statistical analysis, variable correlation research, result visualization, and simple predictive modeling for factors associated with diabetes (DIABETES) and coronary heart disease (CHD). The code comprehensively analyzes the relationships between health behaviors, demographic structure, socioeconomic factors and disease prevalence, and outputs corresponding charts, result tables and cleaned data files as project deliverables.

## Project Content
The project is organized across four core scripts:

- `CDC_scraping.py`: Connects to the CDC PLACES API (Socrata endpoint `cwsq-ngmh`) and pulls county-level health measure data across the entire United States. Filters for 9 key measures including diabetes, coronary heart disease, obesity, smoking, physical inactivity, and mental health. Pivots the data so each measure becomes its own column, indexed by county FIPS code.

- `census_scrape.py`: Connects to the U.S. Census Bureau API and retrieves 2022 American Community Survey (ACS) 5-Year Estimates at the county level. Pulls raw counts for poverty, unemployment, income, education, age, insurance coverage, and race, then derives percentage-based features (e.g. `pct_uninsured`, `pct_poverty`, `pct_bachelors`) for use in modeling.

- `combined_dataset.py`: Merges the CDC and Census datasets on their shared county FIPS code using an inner join, and exports the combined result as `diabetes_heart_factors.csv`. This file serves as the input for the main analysis script.

- `data_cleaning_analysis.py`: Reads `diabetes_heart_factors.csv` and performs the full analysis pipeline — data cleaning, descriptive statistics, correlation analysis, hypothesis testing, visualizations, and random forest predictive modeling for both diabetes and CHD prevalence.

## How to Run

First, generate the dataset by running the data collection scripts:
```bash
python combined_dataset.py
```
Then run the main analysis:
```bash
python data_cleaning_analysis.py
```

## Input File

`combined_dataset.py` automatically generates the required input file:
- `diabetes_heart_factors.csv`

## Main Output Files
After running, some analysis result files will be generated, such as:

  ### Data Tables
  - `cleaned_diabetes_heart_data.csv`: cleaned data
  - `descriptive_statistics.csv`: descriptive statistics results
  - `correlation_results.csv`: correlation analysis results
  - `hypothesis_testing_results.csv`: hypothesis testing results
  - `diabetes_feature_importance.csv`: diabetes model feature importance
  - `chd_feature_importance.csv`: heart disease model feature importance
  - `model_performance_comparison.csv`: model performance comparison
  - `state_disease_rankings.csv`: state-level disease prevalence ranking

  ### Images
  - `correlation_heatmap.png`
  - `disease_distribution.png`
  - `health_factors_boxplot.png`
  - `income_education_disease.png`
  - `scatter_plots.png`
  - `diabetes_residual_analysis.png`
  - `chd_residual_analysis.png`
  - `feature_importance_comparison.png`

## File Description
- `CDC_scraping.py`: CDC PLACES API scraper, outputs county-level health measures
- `census_scrape.py`: Census ACS API scraper, outputs county-level demographic and socioeconomic features
- `combined_dataset.py`: Merges CDC and Census data into the final dataset
- `data_cleaning_analysis.py`: main analysis script
- `diabetes_heart_factors.csv`: original dataset (generated by `combined_dataset.py`)
- Other `.csv` and `.png` files: result files generated after running the analysis script

## Notes
- Run `combined_dataset.py` before `data_cleaning_analysis.py` to ensure the input dataset exists
- Make sure all scripts are in the same directory before running
- If required dependency libraries are missing, the scripts will not run properly
